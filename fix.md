修复总结
- 根因：LVGL API 分别在多个任务中被调用（tick_task 和音频任务），导致非 LVGL 线程里刷新时间过长，触发看门狗。
- 处理思路：音频任务只通过消息队列上报 UI 数据，LVGL 线程定时消费并刷新界面。

修改点
- main/ns4168/audio.c
  - 新增 UI 消息队列与消息类型，发送时间/进度/歌词更新。
  - 移除音频任务中的直接 LVGL 调用。
- main/ns4168/audio.h
  - 声明 UI 消息结构体与取消息接口。
- main/main.c
  - 不再使用 tick_task。
  - 在 LVGL 任务内创建定时器回调，消费消息并调用 ui_tick()。

结果
- 所有 LVGL 调用统一在 LVGL 任务中执行。
- 看门狗异常不再出现。
